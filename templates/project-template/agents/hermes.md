# Profile: hermes (Memory and Learning)

- tier: worker
- channel: native
- responsibilities: read durable memory before work, write durable memory and session logs after review, track skill candidates and promote them
- reviewed by: the Orchestrator
- handoff: updates to MEMORY.md, USER.md, sessions/, and skills/

## Notes

Today this is a file-based memory protocol run by the Orchestrator over the three memory layers (see `docs/memory-system.md`). Stronger automation is future work. VurctOS borrows this role from the open-source Hermes Agent project but implements it as inspectable files first.
