# Manifesto

We are tired of copy-pasting between AI tools.

We are tired of explaining the same project again and again.

We are tired of losing decisions inside chat histories, browser tabs, exports, screenshots, and random folders.

AI creators do not need another generic agent framework. They need an AI team.

Not one chatbot.

Not one magic prompt.

Not one closed workspace that traps the work.

They need a local operating layer that remembers the project, coordinates the tools, and learns the workflow.

## We Believe Creators Need An AI Team

ChatGPT, Claude, Codex, Gemini, Kling, Runway, ComfyUI, and editing tools all have strengths.

The problem is not that the tools are weak. The problem is that the tools are disconnected.

The creator becomes the bridge. The creator copies context, reformats prompts, remembers rules, tracks versions, names outputs, judges quality, and rebuilds state every time a tool changes.

VurctOS should make that bridge real.

## We Do Not Want Another Generic Agent Framework

Generic frameworks often start with abstractions before proving a workflow.

VurctOS starts with a creator problem:

How do we turn one input video into a useful creative prompt pack without losing context?

The first answer is simple:

- project folders
- markdown files
- workflows
- memory
- skills
- subscription-first handoff

Automation comes after the workflow is proven.

## We Believe AI Should Learn Workflows

AI should not only answer prompts. It should learn how the user works.

If a creator repeatedly analyzes videos, extracts frames, identifies hooks, writes Kling prompts, reviews outputs, and packages final prompts, that pattern should become reusable.

The system should notice repetition, capture it, and turn it into a skill.

## We Believe Memory Should Capture Decisions

Most memory systems remember facts.

Creators need more.

They need memory that captures taste, direction, constraints, rejection patterns, tool behavior, visual standards, and production decisions.

The question is not only "what did the user say?"

The better question is "how does this user decide?"

## We Believe In Subscription-First AI

Many creators already pay for AI tools. The first version should use that reality instead of forcing every workflow through paid APIs.

Official login workflows, local files, and copy-ready prompt packs are enough to begin.

API integrations can come later.

## We Believe The Future Is Personal AI Operating Systems

The next step after chatbots is personal AI operating systems.

An operating system for AI work does not have to own every model. It has to coordinate the work, remember the context, and help the user improve over time.

VurctOS starts with AI video production because that workflow is painful, multi-tool, creative, and highly repeatable.

The ambition is large.

The first version is practical.
